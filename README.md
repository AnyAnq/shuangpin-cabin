# 双拼小筑 Shuangpin Cabin

一个面向中文输入肌肉记忆训练的沉浸式双拼练习网站。

双拼小筑不是简单的打字计时器。它围绕“双拼编码、输入节奏、错因修正和连续练习”重新组织练习流程，让用户在低压、专注、连续的环境里，把常用字词、易错键位和真实输入场景练成稳定手感。

## 项目定位

双拼学习最难的部分，往往不是记住规则，而是把规则变成不需要思考的反射。

本项目希望解决三个问题：

- **练什么**：提供诗词句子、绕口令、词库练习和易错复练，而不是让用户机械地敲单字。
- **怎么练**：每个字下方显示双拼编码，引导用户建立“字形 -> 音节 -> 键位”的稳定路径。
- **错了怎么办**：按错不会只给一个红色反馈，而是沉淀为错题记录，再通过错因分组复练修正错误肌肉记忆。

## 核心能力

### 沉浸式练习

- 支持 **小鹤双拼** 与 **自然码** 一键切换。
- 支持诗词句子、绕口令、词库练习、易错练习四类模块。
- 练习过程中实时统计准确率、用时、最大连击数和 WPM。
- 顶部进度条随输入动态推进，减少额外操作打断。

### 逐字编码引导

- 每个可练汉字下方显示对应双拼编码。
- 当前编码以键帽形式提示，帮助新手建立键位节奏。
- 可在设置中关闭逐字编码，让熟练用户进入更纯粹的盲打状态。

### 错因分组复练

- 错题记录会保存目标字、正确编码、期望键、误按键、错误类型、错误次数和连续正确次数。
- 易错练习不再孤立复练单字，而是自动生成同错因分组。
- 记录页提供“纠错教练”视角，优先指出当前最该修正的问题。

## 设计语言

双拼小筑采用暖色、低对比、轻拟物的视觉方向。界面尽量把注意力留给目标文字和键位反馈，侧栏只承担入口，右侧面板只提供必要的状态信息。

整体交互原则：

- 不用传统输入框打断注意力。
- 不用频繁弹窗切断练习节奏。
- 不把统计面板做成冷冰冰的数据后台。
- 把错误反馈设计成“下一步怎么练”，而不是单纯责备用户按错。

## 内容来源

当前练习内容分为三类：

- **在线内容**：诗词句子、绕口令、词库。
- **本地数据**：练习记录、错题记录、偏好设置和已安装词库保存在浏览器 IndexedDB。
- **词库内容**：词库安装后保存在浏览器本地，用于连续的场景化练习。

## 自定义词库

用户可以制作自己的本地词库，并通过设置面板导入练习。本地词库和在线词库会分开管理；进入词库练习后，可以选择某一个词库单独练，也可以点击 **混合**，随机混合当前已安装的全部词库。

### 标准 JSON 格式

JSON 词库必须是一个词库包对象，不能只写词条数组。推荐文件名示例：`my-tech-vocabulary.json`。

```json
{
  "schemaVersion": 1,
  "id": "my-tech-vocabulary",
  "name": "技术术语词库",
  "version": "1.0.0",
  "author": "你的名字",
  "license": "Personal",
  "pricingType": "owned",
  "description": "用于练习编程和软件开发常见术语。",
  "tags": ["custom", "tech"],
  "entries": [
    { "text": "字符串", "weight": 100, "tags": ["编程"] },
    { "text": "初始化", "weight": 99, "tags": ["编程"] },
    { "text": "数组", "weight": 98, "tags": ["数据结构"] }
  ]
}
```

字段说明：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `schemaVersion` | 是 | 当前固定为 `1`。 |
| `id` | 是 | 词库唯一 ID。再次导入同 ID 的 JSON 词库会覆盖旧版本。 |
| `name` | 是 | 词库显示名称。 |
| `version` | 是 | 词库版本号，例如 `1.0.0`。 |
| `author` | 是 | 作者或来源。 |
| `entries` | 是 | 词条数组。每个词条至少需要 `text`。 |
| `license` | 否 | 授权说明，默认 `Personal`。 |
| `pricingType` | 否 | 可选值：`owned`、`free`、`paid`，默认 `owned`。 |
| `description` | 否 | 词库描述，默认 `从本地文件导入的自定义词库`。 |
| `tags` | 否 | 词库标签数组，默认 `["custom", "local"]`。 |

词条字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `text` | 是 | 要练习的词。必须是纯中文，最长 12 个汉字。 |
| `weight` | 否 | 权重，数字越大越优先出现在前面的练习组。 |
| `tags` | 否 | 词条标签数组。 |
| `source` | 否 | 词条来源说明。 |

导入时会自动过滤不符合规则的词条：空词、超过 12 个汉字、含英文、数字或符号的词条不会进入练习。

### TXT / CSV 快速格式

如果只是自己练习，最简单的方式是创建 `.txt` 文件，一行一个词：

```text
字符串
初始化
数组
加载
客户端
应用程序
```

也可以使用 `.csv` 文件写权重和标签：

```csv
text,weight,tags
字符串,100,编程/基础
初始化,99,编程
数组,98,数据结构
```

TXT/CSV 规则：

- 每行第 1 列是词条文本。
- 第 2 列可以写权重；不写时会自动分配。
- 第 3 列可以写标签，支持用 `/`、空格或 `、` 分隔。
- 空行、以 `#` 或 `//` 开头的注释行会被过滤。
- 重复词条会自动合并，并保留更高的权重。

### 导入教程

1. 启动网站后，点击左侧栏底部的 **设置**。
2. 找到 **词库与项目**。
3. 点击 **导入本地词库**。
4. 选择 `.json`、`.txt` 或 `.csv` 文件。
5. 检查导入预览里的有效词条、重复数量和过滤数量。
6. 可按需修改名称、描述、作者和标签。
7. 点击 **确认导入**。
8. 回到首页选择 **词库练习**，点击对应词库开始练习；如果想混合所有已安装词库，点击词库按钮最左侧的 **混合**。

导入后的词库保存在当前浏览器本地，不会上传到远程服务器。更换浏览器、清理浏览器数据或清空已安装词库后，需要重新导入。

## 赞助与永久会员

双拼小筑的付款定位是 **赞助支持项目**，不是购买词库内容。赞助满 10 元可获赠永久会员兑换码，兑换后当前浏览器永久有效，用于解锁官方在线词库的安装和更新服务。

免费范围：

- 本地词库导入、练习和导出永久免费。
- 诗词句子、绕口令、每日一言等在线练习内容永久免费。
- 已安装到浏览器本地的词库可以继续用于练习。

赞助流程：

1. 在词库中心点击付费词库上的 **赞助支持**。
2. 使用微信或支付宝扫码赞助。
3. 付款备注中填写邮箱，方便管理员人工核对。
4. 管理员核对到账后，通常 24 小时内生成一次性兑换码并发送给赞助用户。
5. 用户在赞助弹窗输入兑换码，兑换成功后当前浏览器永久有效。

管理员可访问 `/admin/sponsors` 审核赞助记录。达标赞助点击 **生成兑换码**，页面会显示一次性兑换码；未达标但有效的支持可标记为 **普通赞助**，无法核对的记录可驳回。

部署时需要配置 D1 数据库并执行 `migrations/0001_sponsor_membership.sql`。相关环境变量：

```text
GITEE_ACCESS_TOKEN=你的 Gitee 私有仓库 token
ADMIN_EMAILS=你的管理员邮箱
MEMBERSHIP_SPONSOR_THRESHOLD_CNY=10
VITE_MEMBERSHIP_SPONSOR_THRESHOLD_CNY=10
VITE_WECHAT_SPONSOR_QR_IMAGE_URL=/sponsor/wechat.png
VITE_ALIPAY_SPONSOR_QR_IMAGE_URL=/sponsor/alipay.jpg
SESSION_SECRET=生产环境随机密钥
```

用户赞助和兑换流程不需要邮箱验证码，也不需要配置 Resend。`ADMIN_EMAILS` 仍用于识别管理员账号；如果不使用后台页面，也可以直接在 Cloudflare D1 控制台按迁移结构手动写入 `redeem_codes` 记录。

官方词库包应保存在 Gitee 私有仓库中，前端只通过 `/api/vocabularies/...` 请求。未兑换的浏览器下载会员词库会得到 `401` 或 `403`，兑换成功的浏览器会携带本地保存的会员 token，由后端代理读取私有词库包。

## 快速开始

```bash
npm install
npm run dev
```

默认开发地址通常为：

```text
http://127.0.0.1:5173/
```

如果端口被占用，Vite 会自动切换到下一个可用端口。

## 数据与隐私

- 练习记录、错题记录、设置偏好和已安装词库默认保存在用户浏览器本地。
- 项目不会把用户输入历史上传到远程服务器。
- 在线 API 只用于获取练习内容，不参与错题和练习记录存储。
